# sar

## 每秒采集 CPU 使用率
```bash
sar -u 1 3
```
```
Linux 6.5.0 (hostname)   2026年07月12日  _x86_64_  (8 CPU)

10时30分01秒     CPU     %user     %nice   %system   %iowait    %steal     %idle
10时30分02秒     all      3.45      0.00      1.23      0.12      0.00     95.20
10时30分03秒     all      5.67      0.00      1.89      0.05      0.00     92.39
10时30分04秒     all     12.34      0.00      3.45      4.56      0.00     79.65
平均时间:        all      7.15      0.00      2.19      1.58      0.00     89.08
```
> `%iowait` 第 3 秒突增到 4.56% 说明有磁盘 I/O 操作，`%idle` 79.65% 表示 CPU 仍有较大余量。

## 查看内存和 swap 使用
```bash
sar -r 1 3
```
```
10时31分00秒 kbmemfree   kbavail kbmemused  %memused kbbuffers  kbcached  kbcommit   %commit
10时31分01秒   423456   3456789   7744276     61.23    345612   2456789   4567890     34.56
10时31分02秒   421234   3451234   7746498     61.28    345612   2457890   4568000     34.57
```
> `kbavail`（实际可用内存）约 3.4GB 是判断内存是否够用的关键指标，`%commit` 34% 说明无 OOM 风险。

## 查看网络流量统计
```bash
sar -n DEV 1 3
```
```
10时32分00秒     IFACE   rxpck/s   txpck/s    rxkB/s    txkB/s
10时32分01秒        lo      0.00      0.00      0.00      0.00
10时32分01秒     eth0    123.45     56.78     98.76     12.34
10时32分02秒     eth0    234.56    123.45    187.65     45.67
```
> eth0 接收速率 `rxkB/s=187.65` 表示当前下载约 188 KB/s，`rxpck/s` 和 `txpck/s` 显示包速率。

## 查看系统负载和队列长度
```bash
sar -q 1 3
```
```
10时33分00秒   runq-sz  plist-sz   ldavg-1   ldavg-5  ldavg-15   blocked
10时33分01秒         1       234      0.85      1.10      1.23         0
10时33分02秒         3       236      1.50      1.15      1.25         0
```
> `runq-sz` 运行队列长度突然从 1 增加到 3，结合 `ldavg-1` 升到 1.50 说明有进程在争抢 CPU。

## 查看 I/O 和传输速率
```bash
sar -b 1 3
```
```
10时34分00秒       tps      rtps      wtps   bread/s   bwrtn/s
10时34分01秒     45.00     12.00     33.00   1024.00   4096.00
10时34分02秒    123.00     89.00     34.00   8192.00   2048.00
```
> `tps`（每秒 I/O 事务数）从 45 升到 123，`bread/s` 飙升到 8MB/s，说明系统在密集读取磁盘。
