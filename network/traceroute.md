# traceroute

## 基本路由追踪
```bash
traceroute www.example.com
```
```
traceroute to www.example.com (93.184.216.34), 30 hops max, 60 byte packets
 1  _gateway (10.0.1.1)     0.512 ms  0.489 ms  0.502 ms
 2  192.168.100.1 (192.168.100.1)  1.234 ms  1.198 ms  1.221 ms
 3  10.20.30.1 (10.20.30.1)  5.678 ms  5.612 ms  5.701 ms
 4  203.0.113.1 (203.0.113.1)  12.345 ms  12.289 ms  12.401 ms
 5  93.184.216.34 (93.184.216.34)  12.156 ms  12.101 ms  12.189 ms
```
> 每跳发 3 个探测包显示 3 个 RTT，从第 3 跳延迟跃升到 5ms 说明进入运营商骨干网。

## TCP 模式追踪（穿透防火墙）
```bash
sudo traceroute -T -p 80 www.example.com
```
```
traceroute to www.example.com (93.184.216.34), 30 hops max, 60 byte packets
 1  10.0.1.1     0.489 ms  0.512 ms  0.501 ms
 2  192.168.100.1  1.234 ms  1.189 ms  1.298 ms
 3  10.20.30.1    6.123 ms  5.987 ms  6.201 ms
 4  203.0.113.1  11.876 ms 12.103 ms 11.945 ms
 5  93.184.216.34 12.234 ms 12.087 ms 12.156 ms
```
> `-T -p 80` 用 TCP SYN 到 80 端口替代默认 UDP，能绕过大多数防火墙的 UDP 封锁。

## MTR 报告模式（每跳延迟与丢包率）
```bash
mtr -r -c 10 www.example.com
```
```
Start: 2026-07-12T10:00:00+0800
HOST: server01               Loss%   Snt   Last   Avg  Best  Wrst StDev
  1. _gateway (10.0.1.1)      0.0%    10    0.5   0.6   0.4   1.2   0.2
  2. 192.168.100.1            0.0%    10    1.2   1.3   1.1   1.8   0.2
  3. 10.20.30.1               0.0%    10    5.7   5.9   5.4   6.5   0.3
  4. 203.0.113.1             10.0%    10   12.3  12.8  12.0  15.2   0.9
  5. 93.184.216.34            0.0%    10   12.2  12.3  12.0  13.1   0.3
```
> 跳 4 丢包 10% 但终点丢包 0%，说明是中间路由器限速回复 ICMP 而非真正丢包——**只有终点丢包才是真的丢包**。

## 不解析主机名加速追踪
```bash
traceroute -n 8.8.8.8
```
```
traceroute to 8.8.8.8, 30 hops max, 60 byte packets
 1  10.0.1.1     0.523 ms  0.498 ms  0.512 ms
 2  192.168.100.1  1.201 ms  1.234 ms  1.189 ms
 3  10.20.30.1    5.876 ms  5.712 ms  5.901 ms
 4  72.14.237.130  8.234 ms  8.123 ms  8.345 ms
 5  8.8.8.8        8.567 ms  8.432 ms  8.601 ms
```
> `-n` 跳过 DNS 反向解析，输出更快，适合只看 IP 路径的快速排查。
